# swim-ex — P3 (Low Priority)

---

## ISSUE 24 :: Scale test: Bootstrap Storm simulation

**Problem:** Simultaneous cold boots of a large number of nodes against a single seed node can cause process mailbox bottlenecks or race conditions.

**Proposed fix:** Add a test that spawns 63 nodes simultaneously against a single seed node to verify startup storm resilience.

---

## ISSUE 23 :: Scale test: High Latency Jitter and Delay Stress scenario

**Problem:** Current network simulation only tests packet loss (where packets disappear) but does not test delay jitter, which can cause false-positive suspect states that need to be refuted.

**Proposed fix:** Introduce randomized latency delay to the transport simulation to test the robustness of the suspicion and refutation mechanism.

---

## ISSUE 22 :: Scale test: Rolling Upgrade / Redeployment simulation

**Problem:** We do not have a test that validates how the cluster behaves during rolling restarts where nodes boot sequentially with bumped incarnations, which is common in production deployments.

**Proposed fix:** Implement a test simulating rolling upgrades of nodes in batches of 4-8, verifying they successfully rejoin and that the cluster remains stable throughout.

---

## ISSUE 21 :: Scale test: Multi-Way Partition and Heal scenario

**Problem:** The current scale test suite only tests 2-way split-brain scenarios, which does not exercise the protocol's ability to heal more complex topology disruptions.

**Proposed fix:** Implement a test scenario that splits a 64-node cluster into 4 distinct groups, stabilizes them, and then heals them back into a single unified cluster.
